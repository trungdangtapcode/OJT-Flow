Problem & Opportunity

Pain Points

  • Healthcare data conversion is manual and error-prone
  • Non-technical users struggle with JSON, YAML, CSV structures
  • LLM-only systems produce plausible but invalid outputs
  • No traceability, approvals, or audit trail in existing tools
  • Schema drift and PHI exposure risks

Our Solution

  • AI middleware for healthcare data collaboration
  • Natural language interface with rule-based execution
  • Human-in-the-loop review for risky transformations
  • RAG-grounded explanations with provenance
  • Full audit trail and governance controls

RAG & Healthcare Retrieval

Production-grade semantic search with trusted healthcare knowledge

Knowledge Sources

Schemas, data dictionaries, governance rules, transformation examples, FHIR profiles, clinical terminologies

Semantic Vector Search

OpenAI text-embedding-3-small (384d) stored in Postgres pgvector. Real cosine similarity — no keyword fallback.

Source-Aware MMR

Maximal Marginal Relevance with source diversity to avoid near-duplicate chunks from one document.

Optional Reranking

CrossEncoder (BAAI/bge-reranker-base) second-stage reranking when GPU is available.

Graph Handoff

Entity recognition, terminology grounding, FHIR search hooks, and evidence triples for Graph-NER/RAG.

Quality Controls

Freshness gates, integrity checksums, source inventory, regression dashboard, and active-learning queue.

Security & Governance

Threat Model

OWASP LLM Top 10 aligned. Prompt injection defense, output validation, least-privilege MCP tools.

PHI Protection

Sensitive field detection, redaction policies, no raw PHI logging, masking before export.

Audit Trail

Append-only event stream with hash chain integrity verification. Every action is recorded.

Authentication

Google OAuth, HTTP-only session cookies, Postgres session storage, Redis cache, service accounts with API tokens.

Human Review Gates

Semantic changes, low confidence, PHI detection, and destructive ops all pause for human approval.

Deployment Modes

local_dev, demo, pilot, production — each with enforced capability requirements.
